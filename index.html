HTML

  • The head element contains information about the webpage
  • The body element represents the visible content shown to the user
  • HTML stands for "Hypertext Markup Language"
  • HTML is used to create the structure of the website (think of it as the framing of house)
  • As an HTML dev, you are responsible for providing structure and a positive user experience
  • Some HTML elements include:
    1. Head element: a container for meta data
    2. Meta element: contains info about the page that is used by the browser
    3. Charset (character set) element: sets our character encoding
    4. HTTP Equivalent: sets and HTTP header for content
    5. Viewpoint element: sets the default view of our webpage
  • There exists the concept of "parent" and "child" elements
  • Here are a few sections included in the HTML boiler plate
    1. Head
      • Within the head section you have meta
    2. Body
      • Within the body you have a header to start with

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • Repository: aka "repo," is a folder located in GitHub for a project
  • Branch: a workspace to make changes that dont affect the live site
  • Markdown: file formatting
    • Can be considered a basic form of writing and formatting for your prose and code. Can be done on GitHub. ("basic writing and formatting syntax," docs.github.com)
  • Commit Changes: save a new version
  • Pull Request: askng for changes to be pushed from one branch to another
  • Merge: When changes are actually pulled in
  • git hub issue: Task management tool that helps us keep track of our progress. Help's dev's collab on projects and meet deadlines.
  • There are a few branches within Git
    • Main Branch
      • No coding should be done in this branch
      • This is the main branch, the most updated version of the application
    • Feature Branch
      • Work is done in these branches through what is called "commits"
      • It is important that you name each created branch for the feature that will be developed by them to help indicate the purpose of each branch
    • Develop Branch
  • Why create these branches? To create a space to work without affecting the branch that is default
  • Pull request: ask collaborators for feedback on changes
  • git add -A: command that adds modifications in th current working branch to the staging area. The -A flag indicates we want to add all changes
  • git commit: a command that contains a message that describes what the commit contains
    • always write commit messages in present tense
  • -m: flag for move
  • Git push: create a new remte branch or add new changes to an already existing branch
  • git version control: keep track of changes to project

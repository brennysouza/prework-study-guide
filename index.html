HTML

  • The head element contains information about the webpage
  • The body element represents the visible content shown to the user
  • HTML stands for "Hypertext Markup Language"
  • HTML is used to create the structure of the website (think of it as the framing of house)
  • As an HTML dev, you are responsible for providing structure and a positive user experience
  • Some HTML elements include:
    • Head element: a container for meta data
    • Meta element: contains info about the page that is used by the browser
    • Charset (character set) element: sets our character encoding
    • HTTP Equivalent: sets and HTTP header for content
    • Viewpoint element: sets the default view of our webpage
  • There exists the concept of "parent" and "child" elements
  • Here are a few sections included in the HTML boiler plate
    • Head
      • Within the head section you have meta
    • Body
      • Within the body you have a header to start with

CSS

  • CSS makes a website visually appealing
  • CSS = cascading style sheets
  • There are three ways to style a webpage using CSS
    • inline CSS
      • Developers use inline CSS when they want to style an element directly in the HTML file
      • Apply to specific HTML tags
      • Useful for quick and permanent changes
      • Less flexible
    • internal CSS style sheet
      • This contains CSS rules for the webpage in the head section of the HTML file
      • The rules only apply to the page inwhich this element is applied. You can configure CSS classes and IDs.
    • external CSS style sheet
      • This is the most common way to style of a webpage
      • Preferred because it allows developers to keep all their CSS rules in a seperate file, which makes dseign changes easier
      • When using external CSS, we link the file using a link element placed in the HTML's head
  • A margin indicates how much space we want around the outside of an element
  • A padding indicates how much space we want around the content inside an element
  • There are two parts to a CSS element
    • A selector
    • A decleration block
      • A decleration contains two important components:
      • The CSS property we want to apply
      • The value of that property
  • * selector = the asterisk applies rule to all elements on the page
  • Do Not Repeat Yourself Code: Combining two selectors into a single CSS rule
  • A CSS selector will always have a period before it [i.e. ". card"]
  • There are two ways to assign colors in CSS
    • Semantic Refernces [i.e. gray, blue, etc]
    • Hexidecimal code aka Hex #

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • Repository: aka "repo," is a folder located in GitHub for a project
  • Branch: a workspace to make changes that dont affect the live site
  • Markdown: file formatting
    • Can be considered a basic form of writing and formatting for your prose and code. Can be done on GitHub. ("basic writing and formatting syntax," docs.github.com)
  • Commit Changes: save a new version
  • Pull Request: askng for changes to be pushed from one branch to another
  • Merge: When changes are actually pulled in
  • git hub issue: Task management tool that helps us keep track of our progress. Help's dev's collab on projects and meet deadlines.
  • There are a few branches within Git
    • Main Branch
      • No coding should be done in this branch
      • This is the main branch, the most updated version of the application
    • Feature Branch
      • Work is done in these branches through what is called "commits"
      • It is important that you name each created branch for the feature that will be developed by them to help indicate the purpose of each branch
    • Develop Branch
  • Why create these branches? To create a space to work without affecting the branch that is default
  • Pull request: ask collaborators for feedback on changes
  • git add -A: command that adds modifications in th current working branch to the staging area. The -A flag indicates we want to add all changes
  • git commit: a command that contains a message that describes what the commit contains
    • always write commit messages in present tense
  • -m: flag for move
  • Git push: create a new remte branch or add new changes to an already existing branch
  • git version control: keep track of changes to project

JavaScript

  • JS is an object oriented programming language
  • JavsScript is the tool that takes a static HTML/CSS webpage and makes it more interactive
  • Anything from clicking a button, retrieve a button, or fill out a form
  • Basic javascript features include: variables, strings, and data types
  • Control flow: the order in which a computer executes code in a javacript file
  • Using control flow, I can add conditional logic to this project
  • Javascript was invented by Brendan Eich
  • API's: Application Programming Interfaces
  • As JavaScript skills grow, my websites will enter a new dimension of power and creativity
  • Variables: containers that store values
  • Some consider it a good habit to terminate each statement with a semicolon
  • Shouldn't put a semicolon after a closing curly bracket
  • Comments: snippets of text [i.e. /*...*/]
  • Operators: a mathematical symbol that produces a result based on two values
  • Conditionals: code structures used to test if an expression returns true or not
  • Functions: packaging functionality that you wish to reuse
  • i.e. of a JavaScript variable: [var shape = "circle";]
    • "shape" is the name of container or variable
    • stored data is circle
  • console.log
    • console refers to a test environment that developers use to check out their code
    • .log() is a method which means a set of instructions that can be executed by a computer
  • console.log was created to make it easier for developers to test & debug code
  • Control flow: the order in which a computer executes code in a script or file
  • Remember that the order of code affects the outcome
    • JavaScript is read from top to bottom, be intentional!
  • Truthy: when a condition is true, it will do whatever action is asked to do in between the curly brackets
  • Falsy: a value such as: 0,-0, On, "", null, undefined, NaN
  • Array: a single variable that is used to hold a group of data typically related
    • i.e. var shape = ["triangle", "square", "pentagon", "circle"];
  • An array can contain a string, a number, and a Boolean value
    • i.e. var studentInfo = ["lu", 54, true];
  • In JavaScript and many other programming languages, arrays are zero-indexed and sequential. This means that the first item in any array has an index of 0, not 1. The indices, or location identifiers, will always be whole numbers that increase by 1 each time.
  • For loop uses the predictable pattern of indices to perform a task on all the items in an array by allowing a single code block to be executed over and over
  • Loops are an efficient way to log numerous items are by using loops
    • for loops: telling computer to go through each of the shapes in array and log them to console until no shapes left
    • for loop contains three statements
    • First statment: starting point is a variable of x [i.e. var x = 0]
    • Second statement: a condition [i.e. x < shapes.length;]
    • Final statement: x+1 or x++, every time the code loops, x will increase by 1
  • A function: a set of instructions that tells the computer how to perform a certain task
